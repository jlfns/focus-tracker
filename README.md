# Focus Tracker · 关注追踪器

> 持久化关注列表管理器 — 把关心的事项存起来，一键批量查询。

## 这是什么

一个 AI 助手 Skill，让你用自然语言管理关注列表。添加想追踪的东西，随时一句话全部查完。

- 📊 **数值查询** — 金价/汇率/加密货币/天气/物流
- 📰 **信息聚合** — 新闻/科技/政策/开源项目/竞品/学术
- 🛒 **行动决策** — 比价/票务/招聘/房产/优惠/课程

## 安装

### WorkBuddy 用户

下载 `focus-tracker.skill` → 打开 WorkBuddy → `/skills` → 导入。

### 其他 AI 助手

将本仓库所有文件放入对应技能的 `references/` 目录，或直接作为提示词使用。搜索指令使用中文自然语言，不绑定特定工具名。

## 使用

| 你说 | 会做什么 |
|------|---------|
| "关注黄金价格" | 分类 → 存入本地列表 |
| "我关注了哪些" | 展示列表 |
| "修改黄金价格为金价" | 更新并重分类 |
| "取消关注黄金" | 模糊匹配删除 |
| "看看我关注的" | 先确认 → 联网批量执行 |
| "清空关注" | 确认后清空 |

数据存储：`{用户主目录}/.workbuddy/focus-tracker/focus-list.json`

## 架构

```
                       ┌──────────────────┐
                       │   用户自然语言输入   │
                       │  "关注黄金价格"     │
                       └────────┬─────────┘
                                │
                       ┌────────▼─────────┐
                       │ ① 关键词分类器      │  ← references/rules.md
                       │   18子类规则匹配    │     零模型调用，纯字符串匹配
                       │   → finance/A类   │     最长关键词优先
                       └────────┬─────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                  ▼
     ┌────────────┐   ┌────────────┐   ┌────────────┐
     │② A: 数值查询│   │② B: 信息聚合│   │② C: 行动决策│
     │  Handler    │   │  Handler    │   │  Handler    │
     └──────┬─────┘   └──────┬─────┘   └──────┬─────┘
            │                │                │
     ┌──────▼─────┐   ┌──────▼─────┐   ┌──────▼─────┐
     │③ 联网搜索   │   │③ 联网搜索   │   │③ 联网搜索   │
     │  多源采集    │   │  多源采集    │   │  多平台比价  │
     └──────┬─────┘   └──────┬─────┘   └──────┬─────┘
            │                │                │
     ┌──────▼─────┐   ┌──────▼─────┐   ┌──────▼─────┐
     │④ 执行验证   │   │④ 执行验证   │   │④ 执行验证   │
     │  纠错输出    │   │  纠错输出    │   │  纠错输出    │
     └────────────┘   └────────────┘   └────────────┘
```

**设计原则**：

- **分类器零开销**：纯关键词规则匹配，不消耗 token，毫秒级响应。混合意图自动拆分（如"黄金和苹果发布会"→ 两个独立项）
- **并发独立执行**：每个关注项独立 Agent，互不共享上下文，互不消耗对方 token
- **结果仅拼接**：聚合层不做语义关联，只做列表拼接
- **策略模式**：三个 Handler 实现统一接口，新增策略组只需加一个 handler 文件
- **扩展性**：新增子类 → `rules.md` 加一行关键词；新增策略组 → 加一个 handler.md。不影响已有逻辑

## 纠错机制

每一类关注项都有专属的纠错验证，确保输出可信。

### A 类 — 数值查询（多源交叉 + 日期新鲜度）

| 步骤 | 内容 |
|------|------|
| 1. 日期检查 | 对比 `data_date` 与当前日期，超 1 天标记为 stale |
| 2. 多源采集 | 至少 2 个独立来源，搜不同关键词交叉验证 |
| 3. 偏差计算 | `|来源值 - 主值| / 主值` |
| 4. 阈值判定 | <5% → ✅ / 5%-10% → ⚠️ / >10% → ❌ |
| 5. 单一来源 | 仅一个来源时降级为 ⚠️ |

**示例**：
```
📊 金价 | 金融行情
数值：$4,507/oz · ¥991/克
来源：百家号($4,507) · mygoldcalc($4,510)
偏差 0.07% → ✅ 多源交叉验证通过
```

### B 类 — 信息聚合（可信度评分 + 一致性检测）

**可信度四级权重**：

| 等级 | 权重 | 典型来源 |
|------|------|---------|
| L1 | 1.0 | 人民日报、新华社、政府公报、GitHub 官方 |
| L2 | 0.8 | 财新、第一财经、36氪、极客公园、交易所公告 |
| L3 | 0.6 | 知乎高赞、微博大V、B站UP主、认证自媒体 |
| L4 | 0.3 | 匿名来源、未验证消息、个人社交媒体 |

**一致性三档**：
- ✅ **一致** — ≥3 个独立来源说法相同
- ⚠️ **有争议** — 多数一致但个别分歧
- ❌ **冲突** — 来源间说法矛盾

**综合判定**：
- 评分 ≥ 0.8 且一致 → ✅ 高可信
- 评分 ≥ 0.6 且有争议 → ⚠️ 需关注分歧点
- 评分 < 0.6 或冲突 → ❌ 不可靠

### C 类 — 行动决策（跨平台比价 + 时效性）

| 步骤 | 内容 |
|------|------|
| 1. 跨平台采集 | 至少 2 个平台比价（京东/淘宝/拼多多 等） |
| 2. 价格偏差 | 最低价与其他平台差 >3% → ⚠️ |
| 3. 时效性 | `updated_at` 超过 24h → ⚠️ 可能已过期 |
| 4. 可用性 | 标记缺货/已下架 → 剔除或降级 ❌ |
| 5. 单一来源 | 仅一个渠道 → ⚠️ 无法交叉验证 |

**优先规则**：当硬件类关键词（显卡/CPU/GPU/RTX）与价格类关键词（价格/多少钱/买）同时命中 → 自动归为 C 类，走比价流程而非新闻聚合。
